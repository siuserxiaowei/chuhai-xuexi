于是我想到了一个应用场景：

一个领域有大量的长尾关键词，分析需求是个老生常谈的工作，在以前我提到过关键词的自动分类。

但是，如果仅仅只是分析这个领域的需求类别，并不一定要挨个分类，只需要知道类别即可。

从5118下载一份“素材”的数据，可以下载80万条关键词：

![image]()

把这80万条丢给DeepSeek是不可能的：

![image]()

首先是把一整份80万的数据上传之后，它只能读取1%，其次分批次上传的话，有份数限制：

![image]()

而且还会各种报错；

ima不支持txt和Excel，最好的方案是word，word又不能一次性放入几十万行长尾词。

那就只能分成很多份文件再一起上传，一个word放一万个长尾词是没问题的：

![image]()

一个长尾词平均5-10个字，相当于这份word是一份几万个字的文章，还是有点多。

于是我按一份word 1000个长尾词来分割，几千个字一篇文章，这很正常，这是让ai写的代码：

from docx import Document
import math

def split_keywords_to_docx(input_file, lines_per_file=1000, output_prefix='keywords_'):
    # 读取原始文件
    with open(input_file, 'r', encoding='utf-8') as f:
        lines = [line.strip() for line in f if line.strip()]  # 去除空行和前后空格

80万条数据，被分割成800篇word文档，这能上传吗？还真可以：  
![image]()
一次性就可以把这800份文档上传上去，也不是很久，几分钟。

接下来我就提问：  
![image]()
但是很遗憾，干不动，非常慢，而且最终都会出错。  
ima倒不像DeepSeek那样只能分析几千个词，它是真的在分析能分析的文档：  
![image]()
可是这个数据本身就是不正常的，不管这个知识库里面有多少文档，一般的搜索都只会牵扯几份文档里的几段话，ai理解后再柔和一下就可以了。  
现在的问题是几百份文档里的几十万行内容都涉及，ai一下子要处理这么多的数据再做统计，工程量太大。

再于是，我想到一个折中的方法，既能利用AI，也能使统计出来的数据是不受影响的：  
![image]()
如上图，5118的长尾词挖掘工具，有一个按照“长尾词数量”这个指标排序的功能，这个我以前是讲过的。

什么是长尾词数量，假设5118的数据库里对于“素材”这两个字的长尾词总共就10个：

学习卡 4

这是因为，“风景素材大全”这个长尾词包含“风景素材”这个长尾词，也就是“风景素材大全”是“风景素材”的延伸长尾词，是父子关系(风景素材是父)，其他同理。

那么基于这个逻辑，根据网民的搜索习惯，有“风景素材”这样的词，就一定有“风景素材YY”、“XX风景素材”、“XX风景素材YY”这样的词；

那么在了解需求类别的时候，我们只需要“风景素材”，它不管延伸出什么样的长尾词也算是“风景素材”这一类。

所以我们可以在5118先排序：

![image]()

排序完了之后，再下载，结果就是按排序后的：

![image]()

也就是在5118数据库里，素材 这个词的所有大词我们都下载下来了。

然后再选择有“长尾词数量”的，比如大于2(几乎没有长尾词数量的，要嘛是不正常的词，要嘛是挺长的词)：

![image]()

大概有15万(一两亿的长尾词，其实大词也就这些)。

这15万同样每份1000个词分割在150份word文档里，上传给ima，再使用DeepSeek R1模型来提问：

![image]()

![image]()

结果就清楚的梳理出来了，速度可以接受，总算能实现需求；

在结尾它还特别自信的说没有未提及的的内容。

再给它生成脑图：

![image]()

大小类别总共有大几十种。

有了这一步，我们就可以再进一步提问：

![image]()

甚至让它罗列一些具体词也没问题。

这里再体现了借助AI的优势：它能根据词汇做总结给你准确的命名，比如刘德华素材、郭富城素材，它会总结成：明星素材，这是我们用程序轻易做不到的，要自己准备海量的关联数据才行。

后续，关于这个领域的搜索需求类别，你尽可以在这里提问，需要了解其他领域，则另外再建一个独立的知识库；

美中不足的还是数据量的局限，否则我们可以针对性的提问更多的信息，比如一些搜索结构。

学习卡 5

ps：我们的数据减少到15万后给DeepSeek也是不行的，它只能读取5%，换算一下还是8000个左右的长尾词。

5118其实也有做了一点大致的归类：

![image]()

但是粗糙很多，而且不能反复的去提取。

对于大量的关键词数据，借助ima除了提取需求，其实还有很多灵活的运用：

比如“引流”，它的长尾词经常会出现类似这样：怎么从微博引流到公众号

像这样的长尾词，是一种固定的搜索结构，怎么从XX引流到YY，这就说明XX是一个大家经常在引流的地方，YY是一个流量承载的地方。

那么有多少个XX呢？可以在这里提问统计，你只需要告诉它即可。

又比如“引流”这个词的长尾词会有互联网获取流量这个角度的词汇，也会有医学方面的词汇，

让ai帮我们把大量医学方面的词根词缀提取出来，比如：肺、胃、胸腔 等等，这就可以做为数据清洗的否词库。

今天分享的这个操作的意义在于：低成本的实现了借助ai的底层数据来分析相对大量的关键词的可能。
