核心问题：

不会编程的人想要顺利执行这件事情有两个关键：

1：编程环境

电脑上要有编程环境才能运行代码，而很多环境的配置是比较麻烦的。

学习卡 6

2：描述需求

首先我们要知道，既然是完全不懂得编程，那么很多“高大上”的工作我们是做不了的，因为逻辑复杂的需求AI给你的代码可能是错的或者需要改动的，但你并不懂得修改。

比如涉及联网的一些事情(采集数据)、具备复杂逻辑的程序等等，这些都不合适。

我们更适合去完成的是一些单一重复的批量化工作，那么对于我们非技术岗的朋友常见的本地需求场景会有哪些呢？

比如：文本批处理、文件批处理、数据统计/整合/批处理，这些事情的前面加上一个“大量”就可以体现程序的价值了。

别看只有这么三点，我的Python课程里，除了采集和稍微复杂的案例应用之外，基本都是这三点的范围之内；实际上它们足够解决运营营销推广的岗位的朋友们大量的辅助需求。

学习卡 7

特别说明：

有些朋友会被一些AI教程文章误导，在与AI对话的过程中使用一些完全没必要的描述方式。

事实上AI很聪明，不需要你刻意去使用一些提示词模板才能解决问题。

典型如：你现在是编程专家，你很懂技术，我现在需要你给我写一份代码……

这都是废话！

和AI最好的对话方式是什么？依我之见：使用自然语言，给足前提信息，并减少歧义。

如果AI做不到理解自然语言，那它的答案也不值得期待，如果AI有某方面的能力，不需要你刻意提醒它，在被提问的时候，它自己会去调用。

其实和AI对话，最忌讳的是定义它、限制它，至少长远来看是这样的；除非你有明确的需求或者答案本身是标准的。

你应该给它方向，然后通过它的回答不断去纠正，所谓的纠正是反驳它错误的、你不希望看到的；其他的不要干预。

希望本文对于需求描述的逻辑也能给你在其他与AI沟通的场景中作为参考。

学习卡 8

演示案例：

纠正了一下AI对话的逻辑，我们下面通过一个个案例来演示如何让AI来帮助我们完成编程工作。

按照我们案例演示的方式来下达需求，进一步延伸就可以实现很多任务需求。

我推荐chatgpt3.5的模型即可，因为我们所要完成的工作，这个模型足够；国内一些AI模型在编程方面的能力也已经很完善，可以自行选择。

案例1：

在我的电脑E盘的“test”文件夹里有一份TXT文本文档，它里面存放着一堆长尾词数据，一行一个：

![image]()

现在，我希望把首字是数字的长尾词去除，仅保留首字不是任何数字的长尾词，因为我觉得首字是数字的长尾词可能是不正确的，这对我来说是脏数据。

这是典型的文本批处理工作，如果你把长尾词换成其他内容，工作中可能产生的类似需求会有很多，很多是非常个性化的灵活需求。

如果使用Excel，这可能需要公式，Excel处理的数据量也是有限的，百万以上就很困难了；至于RPA，更不合适做这样的工作，代码才是最优方案。

那么如何实现呢？我认为向AI提编程需求，一般要有这样一套需求描述框架：

a：描述场景(给AI前提条件信息)

b：描述具体任务(让AI撰写核心的功能代码)

c：如何交付给你(代码执行结果如何呈现给你，并不必须)

d：声明/说明/要求/限制(一些特别需要说明的注意事项等，并不必须)

我们把当前的案例，挨个套入这个描述框架：

a：描述场景

“在我的电脑E盘的“test”文件夹里有一份TXT文本文档，它的文件名是：抖音长尾词.txt，里面存放着一些长尾词数据，一行一个。”

以上是完整的描述场景！

描述场景的意义是什么？是为了明确操作目标、减少可能的歧义、同时避免修改。

程序是标准的，是非0即1的，它不接受模糊的概念或信息，再加上同样一句话本身就可能产生不同的理解，这就需要我们尽可能的准确描述、减少歧义，避免产出的代码有错误或者不符合需求。

尽可能准确的描述场景，还有其他好处：比如上述描述里的文件名叫什么，如果你不说，程序给你的代码也不会错，但是代码终究要知道具体处理的是哪份文件，你不说，AI就给一个代替名，这就需要你手动再改一下代码里的文件名。

那你也不知道怎么改是正确的对不对，所以尽可能在语言描述里说清楚，这是不会编程的你可以做到的，得到的代码就不需要你去改动了。

所谓的“一行一个”也是同样的意思，你不说，那有没有可能是一行多个，那一行是几个呢？这是没必要给AI产生歧义的。

b：描述具体任务

“请你给我一份Python代码，它针对这份文本文档里的每个长尾词做判断，只要长尾词的首字是任意一个数字，则去除该长尾词，反之则保留。”

“请你给我一份Python代码”这是必须要出现的，它告诉AI使用Python编程语言；其余部分应该准确的描述这个代码的核心功能是在做什么事情。

你并不一定需要像我的原话那样，但是意思要一模一样，让AI可以明白你的诉求(功能逻辑)。

比如：“请你给我一份脚本，对这份文本文档里的每个长尾词挨个判断，如果长尾词的第一个字是数字，随便数字都算，这个长尾词就去掉，其他的长尾词保留下来；请用Python来写。”

这样一段描述，意思是一模一样的，结果也是一样的。

c：如何交付给你

"保留下来的长尾词，请将它们保存在一份新的文本文档，一行一个，文档命名为“筛选结果.txt”，存储在E盘的“test”文件夹里。"

一般而言，我们的程序执行完后会有两种结果：

1：做了改动，程序正常执行完毕。

比如“文件批处理”，你让程序把电脑里的某些文件的名称批量改了，那么程序就真的改了，你会看到本地的文件名变了，然后程序就结束了。

这种情况你不需要再额外告诉AI如何交付给你，因为目标结果是文件名改变，你已经得到结果了。

2：执行完毕之后，有返回新的数据结果。

比如我们当前的例子，我们针对数据做筛选，筛选完之后，是不是有些丢弃有些保留，那么保留下来的数据怎么处理呢？

通常我们可以选择把数据打印出来：

![image]()

上图中，我们的代码让程序打印数字1，它也确实打印在右边弹出的控制台界面了，我们这就看到1了。

所以你选择让程序打印出筛选完保留下来的长尾词也是可以的，这样你就可以在控制台看到一个个长尾词了。

但是长尾词的数据量可能很大，数据可能还有其他用途，我们不一定希望它打印在控制台(控制台显示是临时的，数据量大会卡)，这个时候我们也可以选择将数据存储在一份新的文件。

那么怎么存储、存到的新文件叫什么，放在哪里，就是你需要告诉AI的了，你甚至可以说：长尾词不要一行一个，全部都在一行，每个长尾词中间有逗号隔开，这也是可以的。

d：声明/说明/要求/限制

如果你有特别需要说明的地方或者额外需要代码执行的事情，可以继续在这里追加；反之，这不是必须要做的。

那么这里为了演示，我们额外增加一个要求：

“对于被去除的长尾词，请单独统计个数，最终将被去除的总长尾词数打印出来。”

OK，我们现在将描述框架里的内容整合在一起，注意了：不需要分为1234，不需要有什么小标题提示，不需要刻意空格换行，我们说过：使用自然语言，所以你只需要正常使用标点符号即可：

“在我的电脑E盘的“test”文件夹里有一份TXT文本文档，它的文件名是：抖音长尾词.txt，里面存放着一些长尾词数据，一行一个。请你给我一份Python代码，它针对这份文本文档里的每个长尾词做判断，只要长尾词的首字是任意一个数字，则去除该长尾词，反之则保留。保留下来的长尾词，请将它们保存在一份新的文本文档，一行一个，文档命名为“筛选结果.txt”，存储在E盘的“test”文件夹里。对于被去除的长尾词，请单独统计个数，最终将被去除的总长尾词数打印出来。”

以上就是一段完整的自然描述，我们将其交给ai：

![image]()

下面是AI给我们的代码： 

import os

学习卡 9

文件路径

input_file_path = 'E:/test/抖音长尾词.txt'

output_file_path = 'E:/test/筛选结果.txt'

def filter_and_count_keywords(input_path, output_path):

# 读取长尾词文件

with open(input_path, 'r', encoding='utf-8') as file:

lines = file.readlines()

# 初始化计数器

removed_count = 0

# 过滤首字为数字的长尾词

filtered_lines = []

for line in lines:

if line and line[0].isdigit():

removed_count += 1

else:

filtered_lines.append(line)

# 将过滤后的长尾词写入新文件

with open(output_path, 'w', encoding='utf-8') as file:

file.writelines(filtered_lines)

# 打印被去除的长尾词数量

print(f'被去除的长尾词总数: {removed_count}')

执行过滤操作

filter_and_count_keywords(input_file_path, output_file_path)

#### 案例2：

有了以上的完整演示，接下去我们就不一定演示程序的实现过程了，重在理解描述框架。  
  
在我的电脑E盘的“test”文件夹里有100份Excel文件，我希望将所有文件的文件名后面都加上“-20240520”。  
我们来描述一下这个需求：  
  
a：描述场景  
“在我的电脑E盘的“test”文件夹里有一些Excel文件。”  
PS：我们的场景明明是有100份，为什么这里说有一些呢？因为除非你有明确要求只处理100份，否则我们说有一些，就是没有具体数量，后续再加上“每个”这样的描述，那就不管文件夹里有多少个都算。  
b：描述具体任务  
“请你给我一份Python代码，它可以将里面的每个Excel文件的文件名后面叠加上“-20240520”这段文本。”  
c：如何交付给你  
由于是没有输出或返回的，所以不需要明确交付描述，我们只需要过后看结果。  
d：声明/说明/要求/限制  
“该操作仅针对后缀是“.xlsx”的Excel文件。”  
PS：因为Excel文件也有后缀是“.xls”的类型。  
  
将他们整合一下：  
“在我的电脑E盘的“test”文件夹里有一些Excel文件，请你给我一份Python代码，它可以将里面的每个Excel文件的文件名后面叠加上“-20240520”这段文本。该操作仅针对后缀是“.xlsx”的Excel文件。”
