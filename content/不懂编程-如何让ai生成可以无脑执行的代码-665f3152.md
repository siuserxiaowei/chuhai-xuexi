### 特别说明：

有些朋友会被一些AI教程文章误导，在与AI对话的过程中使用一些完全没必要的描述方式。  
事实上AI很聪明，不需要你刻意去使用一些提示词模板才能解决问题。  
典型如：你现在是编程专家，你很懂技术，我现在需要你给我写一份代码……  
这都是废话！  
  
和AI最好的对话方式是什么？依我之见：使用自然语言，给足前提信息，并减少歧义。  
如果AI做不到理解自然语言，那它的答案也不值得期待，如果AI有某方面的能力，不需要你刻意提醒它，在被提问的时候，它自己会去调用。  
  
其实和AI对话，最忌讳的是定义它、限制它，至少长远来看是这样的；除非你有明确的需求或者答案本身是标准的。  
你应该给它方向，然后通过它的回答不断去纠正，所谓的纠正是反驳它错误的、你不希望看到的；其他的不要干预。  
希望本文对于需求描述的逻辑也能给你在其他与AI沟通的场景中作为参考。  
### 演示案例：

纠正了一下AI对话的逻辑，我们下面通过一个个案例来演示如何让AI来帮助我们完成编程工作。

#### 案例1：

在我的电脑E盘的“test”文件夹里有一份TXT文本文档，它里面存放着一堆长尾词数据，一行一个：  
![image]()
现在，我希望把首字是数字的长尾词去除，仅保留首字不是任何数字的长尾词，因为我觉得首字是数字的长尾词可能是不正确的，这对我来说是脏数据。  
  
这是典型的文本批处理工作，如果你把长尾词换成其他内容，工作中可能产生的类似需求会有很多，很多是非常个性化的灵活需求。  
如果使用Excel，这可能需要公式，Excel处理的数据量也是有限的，百万以上就很困难了；至于RPA，更不合适做这样的工作，代码才是最优方案。  
  
那么如何实现呢？我认为向AI提编程需求，一般要有这样一套需求描述框架：  
a：描述场景(给AI前提条件信息)  
b：描述具体任务(让AI撰写核心的功能代码)  
c：如何交付给你(代码执行结果如何呈现给你，并不必须)  
d：声明/说明/要求/限制(一些特别需要说明的注意事项等，并不必须)  
  
我们把当前的案例，挨个套入这个描述框架：  
a：描述场景  
“在我的电脑E盘的“test”文件夹里有一份TXT文本文档，它的文件名是：抖音长尾词.txt，里面存放着一些长尾词数据，一行一个。”  
  
以上是完整的描述场景！  
描述场景的意义是什么？是为了明确操作目标、减少可能的歧义、同时避免修改。  
程序是标准的，是非0即1的，它不接受模糊的概念或信息，再加上同样一句话本身就可能产生不同的理解，这就需要我们尽可能的准确描述、减少歧义，避免产出的代码有错误或者不符合需求。  
  
尽可能准确的描述场景，还有其他好处：比如上述描述里的文件名叫什么，如果你不说，程序给你的代码也不会错，但是代码终究要知道具体处理的是哪份文件，你不说，AI就给一个代替名，这就需要你手动再改一下代码里的文件名。  
那你也不知道怎么改是正确的对不对，所以尽可能在语言描述里说清楚，这是不会编程的你可以做到的，得到的代码就不需要你去改动了。  
所谓的“一行一个”也是同样的意思，你不说，那有没有可能是一行多个，那一行是几个呢？这是没必要给AI产生歧义的。  
  
b：描述具体任务  
“请你给我一份Python代码，它针对这份文本文档里的每个长尾词做判断，只要长尾词的首字是任意一个数字，则去除该长尾词，反之则保留。”  
  
“请你给我一份Python代码”这是必须要出现的，它告诉AI使用Python编程语言；其余部分应该准确的描述这个代码的核心功能是在做什么事情。  
你并不一定需要像我的原话那样，但是意思要一模一样，让AI可以明白你的诉求(功能逻辑)。  
  
比如：“请你给我一份脚本，对这份文本文档里的每个长尾词挨个判断，如果长尾词的第一个字是数字，随便数字都算，这个长尾词就去掉，其他的长尾词保留下来；请用Python来写。”  
这样一段描述，意思是一模一样的，结果也是一样的。  
  
c：如何交付给你  
"保留下来的长尾词，请将它们保存在一份新的文本文档，一行一个，文档命名为“筛选结果.txt”，存储在E盘的“test”文件夹里。"  
  
一般而言，我们的程序执行完后会有两种结果：  
1：做了改动，程序正常执行完毕。  
比如“文件批处理”，你让程序把电脑里的某些文件的名称批量改了，那么程序就真的改了，你会看到本地的文件名变了，然后程序就结束了。  
这种情况你不需要再额外告诉AI如何交付给你，因为目标结果是文件名改变，你已经得到结果了。  
2：执行完毕之后，有返回新的数据结果。  
比如我们当前的例子，我们针对数据做筛选，筛选完之后，是不是有些丢弃有些保留，那么保留下来的数据怎么处理呢？  
通常我们可以选择把数据打印出来：  
![image]()
上图中，我们的代码让程序打印数字1，它也确实打印在右边弹出的控制台界面了，我们这就看到1了。  
所以你选择让程序打印出筛选完保留下来的长尾词也是可以的，这样你就可以在控制台看到一个个长尾词了。  
  
但是长尾词的数据量可能很大，数据可能还有其他用途，我们不一定希望它打印在控制台(控制台显示是临时的，数据量大会卡)，这个时候我们也可以选择将数据存储在一份新的文件。  
那么怎么存储、存到的新文件叫什么，放在哪里，就是你需要告诉AI的了，你甚至可以说：长尾词不要一行一个，全部都在一行，每个长尾词中间有逗号隔开，这也是可以的。  
  
d：声明/说明/要求/限制  
如果你有特别需要说明的地方或者额外需要代码执行的事情，可以继续在这里追加；反之，这不是必须要做的。  
  
那么这里为了演示，我们额外增加一个要求：  
“对于被去除的长尾词，请单独统计个数，最终将被去除的总长尾词数打印出来。”  
  
OK，我们现在将描述框架里的内容整合在一起，注意了：不需要分为1234，不需要有什么小标题提示，不需要刻意空格换行，我们说过：使用自然语言，所以你只需要正常使用标点符号即可：  
  
“在我的电脑E盘的“test”文件夹里有一份TXT文本文档，它的文件名是：抖音长尾词.txt，里面存放着一些长尾词数据，一行一个。请你给我一份Python代码，它针对这份文本文档里的每个长尾词做判断，只要长尾词的首字是任意一个数字，则去除该长尾词，反之则保留。保留下来的长尾词，请将它们保存在一份新的文本文档，一行一个，文档命名为“筛选结果.txt”，存储在E盘的“test”文件夹里。对于被去除的长尾词，请单独统计个数，最终将被去除的总长尾词数打印出来。”  
  
以上就是一段完整的自然描述，我们将其交给ai：  
![image]()
下面是AI给我们的代码：  

```python
import os

# 文件路径
input_file_path = 'E:/test/抖音长尾词.txt'
output_file_path = 'E:/test/筛选结果.txt'

def filter_and_count_keywords(input_path, output_path):
    # 读取长尾词文件
    with open(input_path, 'r', encoding='utf-8') as file:
        lines = file.readlines()
    
    # 初始化计数器
    removed_count = 0
    
    # 过滤首字为数字的长尾词
    filtered_lines = []
    for line in lines:
        if line and line[0].isdigit():
            removed_count += 1
        else:
            filtered_lines.append(line)
    
    # 将过滤后的长尾词写入新文件
    with open(output_path, 'w', encoding='utf-8') as file:
        file.writelines(filtered_lines)
    
    # 打印被去除的长尾词数量
    print(f'被去除的长尾词总数: {removed_count}')

# 执行过滤操作
filter_and_count_keywords(input_file_path, output_file_path)
```

#### 案例2：

有了以上的完整演示，接下去我们就不一定演示程序的实现过程了，重在理解描述框架。  
  
在我的电脑E盘的“test”文件夹里有100份Excel文件，我希望将所有文件的文件名后面都加上“-20240520”。  
我们来描述一下这个需求：  
  
a：描述场景  
“在我的电脑E盘的“test”文件夹里有一些Excel文件。”  
PS：我们的场景明明是有100份，为什么这里说有一些呢？因为除非你有明确要求只处理100份，否则我们说有一些，就是没有具体数量，后续再加上“每个”这样的描述，那就不管文件夹里有多少个都算。  
b：描述具体任务  
“请你给我一份Python代码，它可以将里面的每个Excel文件的文件名后面叠加上“-20240520”这段文本。”  
c：如何交付给你  
由于是没有输出或返回的，所以不需要明确交付描述，我们只需要过后看结果。  
d：声明/说明/要求/限制  
“该操作仅针对后缀是“.xlsx”的Excel文件。”  
PS：因为Excel文件也有后缀是“.xls”的类型。  
  
将他们整合一下：  
“在我的电脑E盘的“test”文件夹里有一些Excel文件，请你给我一份Python代码，它可以将里面的每个Excel文件的文件名后面叠加上“-20240520”这段文本。该操作仅针对后缀是“.xlsx”的Excel文件。”  
下面是AI提供给我们的代码：  

```python
import os

# 定义文件夹路径
folder_path = 'E:/test'

def rename_excel_files(folder):
    # 获取文件夹中所有文件的列表
    files = os.listdir(folder)
    
    # 遍历每个文件
    for file in files:
        # 只处理后缀是.xlsx的文件
        if file.endswith('.xlsx'):
            # 构造新的文件名
            new_name = file.replace('.xlsx', '-20240520.xlsx')
            # 获取完整路径
            old_file_path = os.path.join(folder, file)
            new_file_path = os.path.join(folder, new_name)
            # 重命名文件
            os.rename(old_file_path, new_file_path)
            print(f'文件 "{file}" 已重命名为 "{new_name}"')

# 执行重命名操作
rename_excel_files(folder_path)
```

PS：我们看到代码里有一段注释说明“获取文件夹中所有文件的列表”这样的一段功能代码，这就是因为我们的自然描述让AI明白我们没有数量限制，而是有多少都算；程序很智能、AI更智能。  
代码同样也不需要任何修改，测试执行也是准确无误，这一切都建立在准确描述。  
  
注意：大家看到上述代码里，程序有一段“print”的指令，这个是打印输出控制台的指令。  
在这里代码会做我们要求的任务，但是还会顺带把修改记录打印出来：  
![image]()
但我们明明没有要求，这是为什么呢？  
因为前面也说了，控制台是一种临时显示，是否打印是无关任何其他的，差别仅是你是否能看到某些打印信息。  
所以AI只是比较周到，通过打印让你看到变化，但不打印也可以的，因为打印这个行为没有任何影响所以AI才会擅作主张。  
  
重点：  
有了案例2，那么很多文件批处理的工作就可以延伸了，比如：文件批量移动(从哪个文件夹到哪个文件夹)、批量删除、批量读取打开等等。  
那么再结合案例1，我们又可以延伸更多的功能场景：  
比如文件可以批量处理、文件又可以读取，那么我们就可以针对几十份几百份的文件里的数据一起处理了，比如将他们整合在一起，存储到新文件，或者将他们整合在一起，然后做其他的统计或分析等等。

a：描述场景

所以应该灵活描述，只要确保你的描述可以实现唯一的操作方案。

d：声明/说明/要求/限制  
“请按照周一到周日的递增顺序读取文件数据；每份文件的第一行为表头，合并后的数据只需要一个表头。”  
  
我们把这段描述合并后发给AI，得到代码：  

```python
import pandas as pd
import os

# 定义文件夹路径和文件名列表
folder_path = 'E:/test/sem'
file_names = ['周一.xlsx', '周二.xlsx', '周三.xlsx', '周四.xlsx', '周五.xlsx', '周六.xlsx', '周日.xlsx']
output_file_path = os.path.join(folder_path, '周报.xlsx')

# 初始化一个空列表，用于存储每个文件的数据
all_data = []

# 读取每个Excel文件中的第一个表格，并将数据添加到列表中
for file_name in file_names:
    file_path = os.path.join(folder_path, file_name)
    df = pd.read_excel(file_path, sheet_name=0)  # 读取第一个表格
    all_data.append(df)

# 垂直合并所有数据，并只保留一个表头
combined_data = pd.concat(all_data, ignore_index=True)

# 将合并后的数据写入新的Excel文件
combined_data.to_excel(output_file_path, index=False)

print(f'合并完成，文件已保存到 {output_file_path}')
```

PS：前面的前提描述越清晰，后面的任务描述就越简洁。  
要善于利用“别称”，将一份事物定义一个描述，此后就方便在其他地方去描述它。  
c：如何交付给你  
“将每个时段的消费总和统计结果放入一份新的Excel文件，文件命名为：“时段消耗总和.xlsx”，存储到E盘里的“test\sem”文件夹里。”  
d：声明/说明/要求/限制  
““时段消耗总和.xlsx”的数据格式为：第一列是时段，第二列是时段对应的消费总和数据，第一行是抬头，分别为“时段”和“消费总和”；数据排列顺序按照0点-23点排序。”  
  
下面是AI给我们的代码：  

```python
import pandas as pd
import os

# 定义文件夹路径和文件名列表
folder_path = 'E:/test/sem'
file_names = ['周一.xlsx', '周二.xlsx', '周三.xlsx', '周四.xlsx', '周五.xlsx', '周六.xlsx', '周日.xlsx']
output_file_path = os.path.join(folder_path, '时段消耗总和.xlsx')

# 初始化一个空的DataFrame，用于存储消费总和数据
total_consumption = pd.DataFrame(columns=['时段', '消费总和'])

# 读取每个Excel文件中的第一个表格，并累加每个时段的消费数据
for file_name in file_names:
    file_path = os.path.join(folder_path, file_name)
    df = pd.read_excel(file_path, sheet_name=0)  # 读取第一个表格
    
    if total_consumption.empty:
        # 如果total_consumption是空的，初始化它
        total_consumption['时段'] = df['时段']
        total_consumption['消费总和'] = df['消费']
    else:
        # 否则，将当前表格的数据累加到total_consumption中
        total_consumption['消费总和'] += df['消费']

# 将累加后的数据写入新的Excel文件
total_consumption.to_excel(output_file_path, index=False)

print(f'统计完成，文件已保存到 {output_file_path}')
```

#### 案例4：

在我们的电脑里存放着1000张同一尺寸的图片，比如1000*800；我们希望将所有图片的尺寸缩小一半，怎么描述呢？  
如果仔细理解了上述几个案例，那么这个案例其实是很简单的，你可以尝试先自己描述一下。  
  
  
下面是这个案例的描述示例：  
a：描述场景  
“在我的电脑E盘里的“test\img”文件夹里有一批图片。”  
b：描述具体任务  
“请你给我一份Python代码，它将里面的每一张图片的尺寸在原来基础上缩小一半。”  
c：如何交付给你  
“将缩小后的图片按照原来的文件名存储到E盘里的“test\img\new”文件夹里。”  
d：声明/说明/要求/限制  
“原来E盘里的“test\img”文件夹里的原图保持不变，不要有任何改动。”  
站在编程角度，这句话不说也是可以的，但是考虑到你以后可能会操作的更多其他任务，最好要有这种保护原始数据的意识。  
  
下面是AI给的代码：  

```python
import os
from PIL import Image

# 定义文件夹路径
source_folder = 'E:/test/img'
destination_folder = 'E:/test/img/new'

# 如果目标文件夹不存在，创建它
if not os.path.exists(destination_folder):
    os.makedirs(destination_folder)

# 遍历源文件夹中的所有文件
for file_name in os.listdir(source_folder):
    source_file_path = os.path.join(source_folder, file_name)
    
    # 检查是否是文件且扩展名为图片格式（可以根据需要增加其他格式）
    if os.path.isfile(source_file_path) and file_name.lower().endswith(('.png', '.jpg', '.jpeg', '.bmp', '.gif')):
        # 打开图片
        with Image.open(source_file_path) as img:
            # 获取原始尺寸
            original_size = img.size
            # 计算新尺寸
            new_size = (original_size[0] // 2, original_size[1] // 2)
            # 缩小图片
            resized_img = img.resize(new_size, Image.ANTIALIAS)
            # 构造目标文件路径
            destination_file_path = os.path.join(destination_folder, file_name)
            # 保存缩小后的图片
            resized_img.save(destination_file_path)

print('所有图片已成功缩小并保存到新文件夹中。')
```

这个需求描述很简单，返回的代码看起来很复杂，但是别管它；经测试，没有任何问题。  
实际上，同样是图片尺寸批量修改这个需求，你可能会有各种个性化的细节要求，比如你只修改“.png”后缀的图片、你可能想把缩小结果直接替换原图等等，那么就需要你自行灵活的去修改需求描述。  
  
图片尺寸可以批量自动修改，图片是不是还可以批量加水印、批量改色值、批量裁剪、批量改格式等等，这些通过准确的描述都可以实现。  
图片想要批量添加水印，那应该怎么说？是不是我的电脑有一堆图片、我在哪里有一张水印、我希望把这张水印批量贴在每一张图片的哪个坐标位置…… 等等，这样的需求你现在应该就知道大致怎么写了。  
  
由此延伸出来的音视频处理：  
我想把每个视频的前10秒分割出来、我想把每个视频里的水印遮住，水印的位置坐标是……、我想把某个片段批量插入到每个视频的最后作为片尾…… 等等；

如果你希望能更好的运用AI这个工具为你服务(仅编程范围)，你可以学习一点基础的编程知识，这并不是要你去学习编程，撰写代码；  
而是了解基础的编程知识可以让你知道：编程可以做什么、不可以做什么；基于此，你才能更好的描述你的任务需求，同时才能在遇到一些问题时懂得选择程序来解决，因为你会知道这件事情编程可以做。  
学习简单的编程知识不需要花钱买课，网上找一份Python入门教程，看看前面的基础知识即可。
