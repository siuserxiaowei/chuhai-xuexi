081501  
作为VIP会员第一篇正式的专属文章，这篇文章我们来聊聊竞价项目挖掘这个问题。  
市面上有一些类似的教程或方法，包括一些所谓的竞价冷门项目挖掘，一般都会有侧重点；  
那么这篇文章所介绍的方式最突出的是量级和低成本：通过这个方式，你可以瞬间以极低的成本获取到约：500W条竞价记录、成千上万无法估算的业务。  
它来自于一个挺有意思的思路，也是我之前反复提到的：要留意细节、要积累基础、要抓住特征。  
### 逻辑思路

#### 信息一

![image]()
5118这个功能，我在过往的分享当中反复出现过几次。  
如果有一个网址在投放竞价，那么通过这个功能去查询主域名就会看到这个域名下投放了多少关键词，以及每个关键词对应的竞价标题(也就是业务)。  
  
它的原理很简单：  
5118每天向百度查询各种关键词，每个关键词查下去都有机会看到一些竞价，这就产生了：1个关键词 to n个网址 这样的一张数据表，这是表1。  
不同的关键词查询下去有可能会出现一样的网址，因为一个网址会投放多个不同的关键词。  
那么积累了很多这样的数据之后，以网址(主域名)为维度，就可以反向透视出一张：1个网址 to n个关键词 这样的一张数据表，这是表2。  
这就是这个功能的原理逻辑。  
  
也就是说：如果你知道一个在投竞价的网址，你拿它在这里查询一下，就可以看到这个网址在过往投放过什么词、上过什么业务(通过竞价标题和落地页链接，一个主域名下可能有不同的页面用于不同的业务投放)  
#### 信息二

在以前，投放竞价都是由广告主自行搭建落地页，所以你要买服务器、要上线一个站点、做一套网站或者一个单页。  
后来，信息流广告出现了；在移动端，需要更合适的页面体验，由此出现了不少H5建站的平台，比如：快站  
![image]()
百度也是有自己的建站平台的，那时候我刚好在代理商，经常用这个平台给客户做落地页。  
![image]()
它应该就是现在的基木鱼的前身。  
  
那么这种建站平台有一个特点，注意了，特征来了：由这种建站工具创建出来的落地页，它们共用服务器，你不需要自己准备服务器，生成的页面URL可以直接访问，所以自然的：它们都有一个共同的主域名！  
比如百度竞价落地页最典型的主域名：[isite.baidu.com](https://isite.baidu.com/)  
  
显然，通过以上两个信息，你已经知道要干嘛了：  
![image]()
有大量的广告主会使用百度提供的落地页来投放广告，所以这个域名就成为极为特殊的存在。  
对于5118来说，他们依然把[isite.baidu.com](https://isite.baidu.com/)当做某家公司的竞价域名，每次查询某个关键词都很可能看到[isite.baidu.com](https://isite.baidu.com/)，然后把这个关键词作为[isite.baidu.com](https://isite.baidu.com/)这个域名下的投放记录。  
然而实际上这是成千上万的广告主组成的。  
  
这是你几秒钟就可以得到的信息，接近200万条竞价广告就在你眼前。  
同时，这是百度为广告主提供的服务，并不限定什么行业，理论上，各种业务都可能在里面。  
而且，这还是5118沉淀了很多年的数据，这么些年出现的业务，你都有机会看到，光用于学习都是非常宝贵的资源。  
  
现在来看，我反复提到过的：平时要留意细节、积累基础、要善于抓住特征，这是非常重要的。  
就这个思路，你已经不需要再看市面上任何的竞价挖掘教程，甚至包装一下都可以拿来当教程卖了，这是巨大的信息差。  
### 思路拓展

不满足，才会进步，思路是好思路，有没有可能再进一步？  
我们上述提到：有很多H5建站平台，那么有没有可能也有很多人选择百度之外的其他建站平台来做落地页投放？  
如果是这样，那就还存在一些不同于[isite.baidu.com](https://isite.baidu.com/)的特殊域名，包括百度可能也还有其他主域名的落地页？  
既然有可能，那么现在的问题是如果有，如何找出它们呢？你可以尝试思考一下。  
  
这需要一点逆向思维，我们发现[isite.baidu.com](https://isite.baidu.com/)这个域名有200W条记录，有大量的无行业差别的关键词可以找到它。  
所以，如果有个域名，它和[isite.baidu.com](https://isite.baidu.com/)有一样的特质，也是一个很特殊的域名，那么应该也有很多词可以找到它。  
![image]()
上图是5118的另一个功能，每次输入一个关键词，它可以返回这个关键词在过往查询百度时所有看到过的竞价记录，也就是我们上述提到的表1。  
可以看到上面返回的数据有域名这一列，所以我们只要找到很多关键词来这里查询，就可以发现可能存在的特殊域名，也就是经常会出现的那些。  
  
那么找什么关键词呢？即不能有偏向性，同时这个词还得有竞价记录，没有人投竞价的词没有用。  
这就需要用到我在群里分享过的思路，见《社群历史分享记录》第73条分享，你可以过后再看。  
简单来说，有些词根是必然会有竞价的、而且没有偏向性，比如：多少钱，对吧，这个词延伸的很多长尾词都有人在投广告。  
然后是5118的核心功能：

所以，你只需要找到很多类似“多少钱”这样的词根，然后查询、点击、导出，你就可以得到大量的竞价记录。  
把导出的所有数据整合统计一下(域名出现次数)，就可以看到经常出现的域名会有哪些了。  
怎么找到很多类似“多少钱”这样的词根？可以通过上述73条分享；其实简单可以罗列很多的：报价、费用、收费、城市名称 等等，这都是大量竞价广告必然出现的。

### 借助API接口

可能有的朋友经常用5118，会考虑一个问题：尽管数据有200万条，但是实际上平台会员单次只能下载60万条。  
这就很尴尬。  
  
索性，5118提供API接口，[https://www.5118.com/apistore/detail/8af3d6ed-2b12-ed11-8da8-e43d1a103141/-1](https://www.5118.com/apistore/detail/8af3d6ed-2b12-ed11-8da8-e43d1a103141/-1:):  
![image]()
通过这个接口，你可以把目标域名下的所有数据全部提取回来，这是我提取回来放到数据库的：  
![image]()
是其中一个表，100块钱可以采集四五百万条数据，很便宜了，下面是我的接口代码：  

```python
import os,requests,time
from pymongo import MongoClient
from bson.objectid import ObjectId

conn = MongoClient('localhost',27017)
db = conn.sem_data

url = 'http://apis.5118.com/bidword/v2'
data = {'url':'isite.wejianzhan.com','page_index':1,'page_size':500,'isc':0}
headers = {"Content-Type":"application/x-www-form-urlencoded; charset=UTF-8",'Authorization':'your api key'}

while True:
    rel = requests.post(url,headers=headers,data=data)
    result_json = rel.json()
    if result_json['errcode'] != '0':
        print(result_json['errcode'],result_json['errmsg'])
        break

    for data_dict in result_json['data']['keywords']:
        insert_data = {
            'keyword':data_dict['keyword'],'title':data_dict['title'],
            'intro':data_dict['intro'],'bidword_semprice':data_dict['bidword_semprice'],
            'bidword_pcpv':data_dict['bidword_pcpv'],
            'bidword_wisepv':data_dict['bidword_wisepv'],
            'bidword_kwc':data_dict['bidword_kwc'],
            'index':data_dict['index'],
            'mobile_index':data_dict['mobile_index'],
            'urlcount_30day':data_dict['urlcount_30day'],
            'urlcount':data_dict['urlcount'],
            'firstfindtime':data_dict['firstfindtime'],
            'joindate':data_dict['joindate'],
            'bidword_recommend_price_avg':data_dict['bidword_recommend_price_avg']
        }
        db.isite.insert_one(insert_data)
        
    time.sleep(1)
    print(result_json['data']['page_count'],result_json['data']['page_index'])
    data['page_index'] += 1
```

当然，有些域名查询下去根本都没有60万条，就不需要通过接口了，直接办会员下载，而有些域名虽然数据量很大，但是我觉得不是我需要的。  
比如这个[b2b.baidu.com](https://b2b.baidu.com/)，如其名，它专门是一些b2b的采购广告，传统行业。

### 小结

通过思路，配合一点技术，可以很轻松的得到大量的广告数据，这个信息是很多人他看不到的。  
从上到下看起来，几乎没有什么成本，无非就是会员费和一点采集费用，相比能得到的数据几乎可以忽略。  
  
数据有了，千万不能只是存在硬盘养灰尘，这就没意义了；  
下面给大家一些操作角度，尽可能帮大家减少数据噪音，以更低的成本去寻找数据的价值。  
### 清洗过滤

#### 去重

首先是去重，去重根据核心维度，我们既然要了解各种不同的业务，那就应该是以竞价标题为主。  
当然，同一个业务的创意可能会有很多种，这没办法直接通过去重处理，我们可以通过其他手段。  
这一步就先去除标题完全一样的。  
#### 时间筛选

其次是按时间维度：  
![image]()
解释一下这两个字段：“发现时间”指的是5118的程序在百度搜索某个关键词之后第一次看到某条广告的时间；  
“最后发现时间”指的是上一次再次搜索这个关键词的时候看到这条广告的时间。  
通过“发现时间”以及截止到我们看到时，我们可以判断这个广告至少投放了多久了。  
而理论上来说，如果一个广告的“最后发现时间”是一年前甚至更久，显然这条广告应该是不在了，要嘛就是这个词他们不投了，要嘛就是这个广告主已经不投了，这个业务不做了。  
  
但是这又涉及到一个问题，5118的数据库里有海量的各种词，那么多久重新遍历一次？  
有没有可能是5118太久没有重新去查询一次这个词进而更新，实际上人家还在做呢？有这种可能。  
那么我目前的建议是：保守一点选择在一年以内的数据优先查看。  
当然一年往上的广告也不是说没有价值了，有各种各样的原因导致这种情况，同时也可能这个业务在这个渠道做不了，但是在别的渠道换一种形式继续在做着，这种情况还不少。  
#### 创建否词库

有很多业务你知道你是不会考虑的，比如丰减壮、黑五类、热门业务等等；  
既然已经确定你不考虑这些业务，那就创建一份否词表，但凡标题包含这个词汇的记录，你可以直接去除。  
注意，否词表的创建逻辑是：表里的词，你确定标题有就一定不要，反之，你不确定标题有的情况是不是一定去除的话，这个词你不要轻易加进去。  
比如：减肥这个词你确定它存在就不要这个标题，那你可以加进去，批发这个词你可能是不想看到一些传统行业，所以你加进去，但是有个业务叫：密钥批发，这就可能带来误伤。  
#### 做聚类

通过聚类把大量类似的数据聚集到一起，可以大大的提升信息查阅的效率。  
我之前聊过不少归类、分类、聚类这样的操作，这些操作都有一个共同的场景：只有一列关键词数据。  
但是，我们现在面对的数据不太一样：  
![image]()
可以看到我们的数据里有关键词和竞价标题，原始数据本身就是通过关键词查询到标题，它们是一一对应的关系。  
百度竞价长期基于关键词匹配模式来匹配广告，关于匹配模式：  
![image]()
不懂竞价的朋友也不必深入理解，简单来说，当我们搜索一个关键词，能看到某个广告，大概率这个关键词里至少会有一个词汇是出现在竞价标题里的：  
![image]()
而如果只有一个重复词汇，这个词汇往往是业务核心词，我们可以利用这一关系！比如某行数据：  
竞价关键词：seo入门教程，竞价标题：seo新手学习,0元试听,8-15年行业经验老师实战教学  
分别分词后：  
seo`、`入门`、`教程` 以及 `seo`、`新手`、`学习`、`0元`、`试听`、`15年`、`行业`、`经验`、`老师`、`实战`、`教学  
你看，seo是两者唯一共有的词汇，同时它也是业务核心描述词汇。  
通过这个逻辑，我们就可以把每个不同的竞价标题的“核心词汇”提取出来，用它作为标识，再把标识一样的放到一起排列输出，就实现了聚类：  
![image]()
这种聚类技术成本极低，效果还算不错，即使有些聚集起来不是一个业务，但是大方向还是差不多的。  
当然，这是思路逻辑，演示的是比较理论的情况，实际情况中，可能遇到没有共同词汇的、可能遇到多个共同词汇的。  
没有共同词汇的相对较少，多个共同词汇的，可以按照自定义的策略，比如最多选择两个或三个词汇，比如按照词频排序这些词汇组合成标识文本、比如按照权重来选择某个词作为核心词，同时当做标识等等，甚至你还可以结合竞价描述文案来判断。  
不同策略有好有坏，选择合适自己的即可。  
PS：对技术朋友说的，在输出数据的时候，根据标识所拥有的标题数量降序输出，也就是标识对应的标题数量越多，优先输出它的数据。  
  
通过以上这几个步骤，已经可以帮你过滤很多噪音，方便你轻松的去寻找各种商机信息，接下来我们来说说数据提取。  
### 数据筛选

我们并不是一定要一行一行的看数据，我们可以有偏向性的选择一些特定数据来针对性的了解；  
那么这就需要一些特定词汇，以下是一些角度：  
#### 按平台或产品

比如基于微信可能产生很多商业机会，那么标题包含微信的竞价我们可以根据这两个字去提取出来。  
所以，类似：抖音、头条、小红书，它们是一些平台名称；PS、剪映、Python，它们是一些产品名称。  
#### 按群体或职业

比如：各种人群描述，老人、学生、白领等等；各种岗位名称，程序员、公务员、会计等等。  
#### 按行业或领域

比如：seo是一个行业、AI是一个领域，它们也会延伸很多的商业机会或在线业务。  
#### 按术语或黑话

比如工具类产品，它们经常会涉及很多专有词汇：批量、自动、一键、系统、下载 等等，我们用这种词汇就可以专门去提取软件工具类的相关广告。  
而一些你知道的行业黑话自然也可以精准的提取到目标广告。  
  
其他类似：技能类、兴趣爱好等等，它们也可以展开很多业务，这些与上述几个角度是类似性质的，这里不再展开。  
总之：要有“类别”的概念，包括我之前提到的人名的思路，要带着类别去思考用于提取数据的词根词缀。  
#### 特殊词汇

有很多特殊词汇可以无差别的提取一些很直接的业务，比如：  
![image]()
有些广告会直接留下微信号，这种可以通过正则表达式全部提取出来，研究一下都是些什么样的业务。  
直接留电话的一般就都是些本地上门业务之类的。  
![image]()
有些广告会直接写多少钱，这些带数字+元/块的广告，可能是一些很“直接”的业务，它只要曝光就会有转化。  
![image]()
有些广告会写24小时自助，什么样的广告会写上这样的词汇呢？一般很可能是一些自动交易的服务，这可是别人在自动赚钱的机器。  
所以，所谓的特殊词汇，关键在于你要去思考广告主可能的业务模式、可能的表达方式、可能的描述形式。  
基于这种思路来思考可能的一些特殊词汇，以此快速的提取你更需要的信息。  
### 进一步了解

截止到这里，对于很多经验丰富的竞价老手，其实已经足够了，剩下的他们会自己去寻找数据的价值。  
但是，考虑到还有很多竞价新手、经验没有很多、甚至没有做过竞价的朋友，他们不知道接下去应该做什么；  
所以，下面内容从普适性的角度给大家一些建议，希望帮助大家更好的运用数据。

#### 理解广告

为什么第一步要先把落地页找出来看看，目的是为了真正的理解广告，知道人家这个业务是在做什么。  
光看一个标题可以知道个方向，但是人家业务的运作模式、核心卖点、变现方式等等，这个还是需要通过落地页上的营销文案才能理解。  
只要先理解，你才能确定这个广告是不是要继续了解。  
#### 深入内部

使用这些自助建站的落地页来投放广告的，一般要嘛是留资、要嘛是加微，基本是这两种方式获客。  
![image]()
![image]()
那么我的建议是如果有微信，一定要添加一下，进入对方私域，去感受对方的营销手段，很多的具体价格、变现形式、交易方式往往要在私域才看得清，你得把这些细节摸一遍。  
如果是提供一些产品服务，比如软件或站点，一定要自己体验一下，最好可以小额付费一下看看。  
#### 找出核心竞争

宝宝起名这个业务，你认为核心竞争是什么？  
如果你不懂技术，你可能会认为，需要做这件事情的话，要搭建一个网页、要有一个自动交易收款、自动生成名字的系统。  
如果你懂技术，这些对你来说很简单，你可能会认为，关键在于人家背后有真正的大师对接，可以做好服务、可以做好沟通和咨询、可以做好营销和复购、不怕出什么问题，你没有这样的资源。  
但是，如果这些你都搞得定，你可能会认为这件事情的关键在于放量，把所有核心流量和周边流量甚至是相关的泛流量一起收进来，然后依靠团队和模式去批量营销、长期营销。  
那这么来看，我们很难确定某个项目的核心竞争是什么。  
  
在这里，我给大家的建议是这样(只能算经验之谈)：  
首先要知道核心竞争的意义，我们找出核心竞争是为了知道这件事情能不能做，如果核心竞争刚好是我们可以做到的、甚至是我们的优势，那做这件事情就很容易。  
所以上述三个层次，第一层只是确保你可以上线跑起来，但是上线跑起来不是目的，得赚钱才行；  
而第二层是影响你做好做差，那么从这里开始，这个就可以看做是核心竞争，是你区别于同行以及在外面观望的人的竞争力。  
至于第三层，它决定了做大做小，这是进一步的优化放大，已经是竞争之上的问题了。  
所以，核心竞争就是指外面观望的人轻易做不到、内部普遍同行也不太行的关键点。  
  
其次是不要基于你的局限来思考业务的核心，不要因为你不懂技术，你就认为核心竞争是技术。  
技术搭建是问题、有“正经”资质的大师稳定合作也是问题，那么对于大部分人来说，哪个问题比较难解决。  
你应该基于大部分人要做这件事情的局限！  
  
以上这些都只是我的一点经验之谈，实际上这样的判断题，往往都要深深的结合个人的认知和经验，这里希望给大家一些参考。  
#### 找到更多同行

当你锁定一个广告时，要尽可能去把更多的同行找出来。  
首先是在数据里把同广告的关键词尽可能提取出来：

基于这些关键词去百度关键词规划师拓展更多行业词(这一步非必须，如果数据里的词不够丰富的话)，然后再到5118的查竞价工具，用这些词去查更多广告：  
![image]()
再通过标题和落地页去判断是否同一个业务。  
  
找到更多同行的目的是什么：  
通过不同团队包装出来的广告了解整个业务的全貌、细节，你才能调查得更全面。  
另外，如果你找不到很多同行，那你也知道这意味着什么；所以这是必要的步骤。  
#### 不同渠道对比

有些朋友可能会误会，觉得找竞价广告后续要做竞价，当然不是这样的。  
这里的主要目的是了解有什么业务，至于是不是在竞价上做，诚然：如果可以的话，别人稳定的模式我们照抄是最好的。  
但是也不一定，你竞价薄弱、seo擅长、自媒体擅长，那去用你擅长的方式来经营。  
  
现在还能在竞价上稳定出单的生意，很多时候是依靠信息差的。  
比如你不会到竞价上买几百块一份的教程，但是有的人可能会，对于这些人来说他们就是不懂；  
这个时候在竞价上做这个业务，本质上是把这些不懂的人给筛选了出来。  
那么如果你跑到闲鱼发现上面几块钱的东西在竞价上卖几十块上百块，  
说明这就是基于信息差、说明这个东西利润很大、说明这个团队是找到了信息差而不是你能力不可及的竞争。
